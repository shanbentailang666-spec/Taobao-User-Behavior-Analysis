# 淘宝电商用户行为分析项目（一周速成版）

## 项目背景
独立完成淘宝APP用户行为数据的端到端分析（约1000万条记录），聚焦用户从**点击 → 加购物车 → 支付**的全链路行为，构建转化漏斗、支付RFM用户分层和时间趋势分析，为电商运营提供数据洞察。

## 使用工具
- Power BI Desktop：数据导入、清洗、交互式**全中文**仪表板

## 数据来源
Kaggle 淘宝用户行为数据集（UserBehavior.csv，约1000万行，2014年11-12月真实用户行为数据）

## 已完成内容（第1天）
- 成功导入约1000万行原始数据
- 时间戳转换为标准日期时间格式
- 添加“行为名称”列（点击、收藏、加购物车、支付）
- 数据已加载到Power BI模型

## 项目目标
- 计算用户行为漏斗转化率（点击 → 加车 → 支付）
- 进行支付RFM用户分层（识别高价值用户）
- 分析时间趋势（日期与小时高峰）
- 输出可落地的电商运营建议

## 后续计划（一周项目）
- 第2天：数据检查 + 行为漏斗分析
- 第3-4天：RFM用户分层 + 时间趋势探索
- 第5-6天：构建全中文交互式仪表板
- 第7天：PDF报告 + 项目包装优化

## 项目亮点
- 处理千万级数据
- 全中文仪表板，贴近国内电商场景
- 包含电商核心指标：转化漏斗 + RFM分层
